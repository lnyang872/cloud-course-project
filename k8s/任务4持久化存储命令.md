# 任务 4：Redis PVC 持久化存储部署与验证命令

## 说明

本任务不修改任务 3 的 YAML 文件，单独使用：

```text
task4-redis-pvc.yaml
```

该文件包含：

- `redis-data-pvc`，StorageClass 为 `csi-disk`，容量 `10Gi`；
- 更新后的 `redis` Deployment，将 PVC 挂载到 `/data`；
- Redis 开启 AOF：`--appendonly yes`，确保写入数据落盘。

## 1. 在 CloudShell 创建或上传 YAML

如果已上传文件，直接进入文件所在目录。

如果需要在 CloudShell 手动创建，执行：

```bash
cat > task4-redis-pvc.yaml <<'EOF'
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: redis-data-pvc
  namespace: default
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: csi-disk
  resources:
    requests:
      storage: 10Gi
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: redis
  namespace: default
spec:
  replicas: 1
  selector:
    matchLabels:
      app: redis
  template:
    metadata:
      labels:
        app: redis
    spec:
      containers:
        - name: redis
          image: redis:7-alpine
          imagePullPolicy: IfNotPresent
          ports:
            - containerPort: 6379
          command:
            - redis-server
            - --requirepass
            - redis123456
            - --appendonly
            - "yes"
            - --dir
            - /data
          resources:
            requests:
              cpu: "100m"
              memory: "128Mi"
            limits:
              cpu: "500m"
              memory: "512Mi"
          volumeMounts:
            - name: redis-data
              mountPath: /data
      volumes:
        - name: redis-data
          persistentVolumeClaim:
            claimName: redis-data-pvc
EOF
```

## 2. 应用 PVC 和 Redis Deployment

```bash
kubectl apply -f task4-redis-pvc.yaml
```

## 3. 查看 PVC 状态

```bash
kubectl get pvc
```

等待：

```text
redis-data-pvc   Bound
```

## 4. 查看 Redis Pod

```bash
kubectl get pods -l app=redis -o wide
```

等待 Redis Pod 为：

```text
1/1 Running
```

## 5. 写入测试数据

获取 Redis Pod 名称：

```bash
REDIS_POD=$(kubectl get pod -l app=redis -o jsonpath='{.items[0].metadata.name}')
echo $REDIS_POD
```

写入数据：

```bash
kubectl exec -it $REDIS_POD -- redis-cli -a redis123456 SET testkey "hello"
```

查询确认：

```bash
kubectl exec -it $REDIS_POD -- redis-cli -a redis123456 GET testkey
```

应返回：

```text
hello
```

## 6. 删除 Redis Pod 触发重建

```bash
kubectl delete pod $REDIS_POD
```

等待新 Pod Running：

```bash
kubectl get pods -l app=redis -w
```

看到新 Pod Running 后按 `Ctrl+C` 退出。

## 7. 重建后再次查询数据

重新获取新 Pod 名称：

```bash
REDIS_POD=$(kubectl get pod -l app=redis -o jsonpath='{.items[0].metadata.name}')
echo $REDIS_POD
```

查询数据：

```bash
kubectl exec -it $REDIS_POD -- redis-cli -a redis123456 GET testkey
```

应返回：

```text
hello
```
