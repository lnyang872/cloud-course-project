# A-1 数据清洗：豆瓣电影评分数据集

## 生成文件

```text
douban_cleaning.py
Dockerfile.douban-spark
douban-cleaning-sparkapplication.yaml
```

## 说明

`douban_movies.csv` 大约 39MB，不适合用 Kubernetes ConfigMap 挂载。这里采用将 CSV 和清洗脚本打包进 PySpark 镜像的方式运行 SparkApplication。

## 1. 上传文件到 ECS

在 Windows PowerShell 执行：

```powershell
scp "D:\Desktop\A.Hui\云计算技术\ks\douban_movies.csv" root@139.9.212.208:/root/
scp "D:\Desktop\A.Hui\云计算技术\ks\k8s\douban_cleaning.py" root@139.9.212.208:/root/
scp "D:\Desktop\A.Hui\云计算技术\ks\k8s\Dockerfile.douban-spark" root@139.9.212.208:/root/Dockerfile.douban-spark
```

## 2. 在 ECS 构建数据清洗镜像

在 ECS 执行：

```bash
cd /root
ls douban_movies.csv douban_cleaning.py Dockerfile.douban-spark

docker build -f Dockerfile.douban-spark -t swr.cn-south-1.myhuaweicloud.com/cloud-course/douban-spark:v1 .
docker push swr.cn-south-1.myhuaweicloud.com/cloud-course/douban-spark:v1
```

推送后去 SWR 华南广州区域，将：

```text
cloud-course/douban-spark:v1
```

设置为公开。

## 3. 在 CloudShell 创建 SparkApplication YAML

```bash
cat > douban-cleaning-sparkapplication.yaml <<'EOF'
apiVersion: sparkoperator.k8s.io/v1beta2
kind: SparkApplication
metadata:
  name: douban-cleaning
  namespace: default
spec:
  type: Python
  mode: cluster
  image: swr.cn-south-1.myhuaweicloud.com/cloud-course/douban-spark:v1
  imagePullPolicy: Always
  mainApplicationFile: local:///opt/spark/app/douban_cleaning.py
  sparkVersion: "3.4.3"
  restartPolicy:
    type: Never
  driver:
    cores: 1
    coreLimit: "1200m"
    memory: "1g"
    labels:
      version: "3.4.3"
    serviceAccount: spark
  executor:
    cores: 1
    instances: 2
    memory: "1g"
    labels:
      version: "3.4.3"
EOF
```

## 4. 提交数据清洗作业

```bash
kubectl delete sparkapplication douban-cleaning --ignore-not-found=true
kubectl delete pod douban-cleaning-driver --ignore-not-found=true
kubectl apply -f douban-cleaning-sparkapplication.yaml
```

查看 Pod：

```bash
kubectl get pods -n default -w
```

等：

```text
douban-cleaning-driver   Completed
```

## 5. 查看清洗输出日志

```bash
kubectl logs douban-cleaning-driver -n default
```

日志中应包含：

清洗前行数: xxx
